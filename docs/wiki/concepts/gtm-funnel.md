# GTM Funnel

Harbormill's **front-half go-to-market motion** — how prospects are found, converted, onboarded,
delivered, and kept on retainer. It makes the [[Harbormill Ladder]] *executable*: the Ladder names
the rungs and prices; this names the motion that moves a stranger up them. The operational assets
live in `docs/gtm/`; the canonical 30-day execution plan lives in the session plan file.

## The funnel

```
~50 warm contacts → ~12 intros → ~4–5 paid Loop Audits → 2 projects → 2 retainers
   (25%)              (35%)          (50%)
```

Work backwards from the goal: to land **2 Focused Projects**, sell ~4–5 [[Loop Audit]]s, which need
~12 intro calls, which need a reached **warm list of ~50.** Each stage's conversion rate is the
diagnostic — an underperforming stage is the precise place to intervene (more volume in, or a
tighter pitch), rather than reworking the whole funnel.

## The five stages

1. **GET** — warm-network activation sprint (personalized outreach + a relentless referral ask).
   Cold channels rarely *close* in 30 days; warm does. Site assets (lead magnet, Calendly) are
   *capture*, not *generation*.
2. **CONVERT** — free 30-min intro (make the pain a dollar number, teach the [[Four-Condition Loop
   Test]], sell the [[Loop Audit]]) → audit → fixed-scope project proposal. The audit fee is
   **credited toward the build** to compress the cycle; the audit's Opportunity Report *is* the proposal.
3. **ONBOARD** — a repeatable signed→live SOP on top of [[Per-Client Deployment]]: deposit, kickoff,
   front-loaded access intake, deck deploy (~30–45 min), dates set.
4. **DELIVER** — one workflow, one success metric, ruthless scope. Build in the config/data seams so
   each build makes the next faster (the scalability compounding).
5. **RETAIN** — convert to a productized retainer at delivery, when ROI is fresh. The deck's monthly
   "Value Delivered" surface re-proves ROI every cycle (churn defense) and a new loop ships monthly.

## The wedge

- **Flagship / retainer engine:** the AIOS deck (white-label, retainer-native).
- **Default first build:** **AR / invoice follow-up** — hardest-dollar ROI, perfect
  [[Four-Condition Loop Test]] fit, solo-deliverable. Lead-intake → CRM is the backup hook.
- **Vertical:** agnostic until revenue lands, then niche on whichever segment converted, so
  marketing compounds and acquisition cost drops.

## Why it's scalable

Same flagship (the deck) + same method (the [[Loop Audit]]) + same monthly motion (review ROI → pick
next loop → build → deck reports it) on every client. Reusable engine work flows upstream to the base
template, so each new client is cheaper to serve than the last.
