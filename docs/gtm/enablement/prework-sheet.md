# Before Your Session — 15 Minutes of Setup

We only have an hour together, and we want to spend it on the interesting part — **directing Claude to
build something** — not on installing software. So please do these four things **before** the session.
It takes about 15 minutes. If anything won't work, don't fight it — jot down what happened and tell the
facilitator; some of it is on IT to unlock, not you.

## Do these four things (~15 min)

1. **Install Claude Code.** Your organization may have a specific way to do this — check your calendar
   invite or ask your Claude admin. If you were just told "install it," follow the official installer
   for Windows. *(You should end up able to run `claude` from a terminal.)*
2. **Sign in** with your work Claude account (the Enterprise account your admin set up). If the sign-in
   asks something you don't recognize, stop and ask your admin — don't guess.
3. **Add the VS Code extension.** Open VS Code → Extensions (the squares icon on the left) → search
   **"Claude Code"** → Install.
4. **Say hello.** In VS Code, open a terminal (Terminal menu → New Terminal), start Claude, and type
   **Hello**. ✅ **You're ready when Claude replies.** That's it — leave it there.

## Make sure these are true (ask IT if you're not sure)

These are usually set up for you, but confirm — the session assumes them:

- **VS Code is installed** on the machine you're bringing, and you can install extensions on it.
- **You can reach the project's Azure DevOps** — both **Repos** and **Boards** — for the project we'll
  be working in.
- **`az login` works** on your machine (the Azure CLI is installed and signed in). If you've never run
  it, ask IT or flag it to the facilitator ahead of time.

## Why we ask

The session is a live walk-through: you'll watch the whole flow once, then do it yourself afterward
with the guide we hand you. Arriving with Claude Code already running means the hour goes to the part
that matters — not to six people installing at once.

**Trouble?** Reply to the invite or message the facilitator before the session. Setup snags are normal
and faster to fix beforehand than live.
